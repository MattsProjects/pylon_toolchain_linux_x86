Detailed Description

The CUpdate class can be used to check which output data has been generated by which CRecipe data update operation and what the corresponding input data is.

The pylon recipe uses the concept of updates. An update can consist of the synchronized processing of a defined set of data by one or more vTools at the same time. Some vTools can trigger updates on their own, e.g., when triggered by a preceding update. This causes a chain of updates.

Thread Safety:
The CUpdate class is not thread-safe.
